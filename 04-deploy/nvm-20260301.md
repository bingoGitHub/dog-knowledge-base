### 📦 第一步：安装必要的依赖（可选但推荐）

打开你的 WSL 终端（Ubuntu-24.04），先更新包索引，并安装 `curl`（如果尚未安装），因为 nvm 安装脚本需要它。

```bash

sudo apt update
sudo apt install curl git -y
```

> `git` 不是必须的，但 nvm 在安装某些 Node 版本时可能会用到，一并装上比较稳妥。

### ⚙️ 第二步：下载并运行 nvm 安装脚本

使用 `curl` 从 nvm 官方 GitHub 拉取安装脚本并执行：

```bash

curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
```

> 如果提示 `curl: command not found`，请先执行 `sudo apt install curl -y`。
> 
> 脚本会自动将 nvm 的配置添加到你的 shell 配置文件（`~/.bashrc`、`~/.zshrc` 等）中。

### 🔄 第三步：重新加载 shell 配置

安装完成后，需要让配置生效。你可以：

- **关闭当前终端，重新打开一个新的 WSL 窗口**；或者
    
- 直接运行以下命令重新加载 `~/.bashrc`：
    

```bash

source ~/.bashrc
```

### ✅ 第四步：验证 nvm 安装成功

运行下面的命令，如果输出版本号（例如 `0.39.7`），说明 nvm 安装成功：

```bash

nvm --version
```

如果提示 `command not found`，可能是配置文件没有正确加载。可以手动检查 `~/.bashrc` 文件末尾是否包含了类似下面的内容，如果没有，请手动添加：

```bash

export NVM_DIR="$HOME/.nvm"
[ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh"  # This loads nvm
[ -s "$NVM_DIR/bash_completion" ] && \. "$NVM_DIR/bash_completion"  # This loads nvm bash_completion
```

添加后再次运行 `source ~/.bashrc`。

### 🌐 第五步：使用 nvm 安装 Node.js

nvm 可以安装任意版本的 Node。例如，安装最新的 LTS 版本：

```bash

nvm install --lts
```

安装完成后，nvm 会自动切换到该版本。你也可以安装指定版本，比如 `18.17.0`：

```bash

nvm install 18.17.0
```

### 🔀 第六步：管理已安装的版本

- **查看已安装的 Node 版本**：
    
    ```bash
    
    nvm list
    ```

- **切换到某个版本**：
    
    ```bash
    
    nvm use 18.17.0
    ```

- **设置默认版本**（下次打开终端时自动使用）：
    
    ```bash
    
    nvm alias default 18.17.0
    ```
    

### 📌 第七步：验证 Node 和 npm

运行以下命令，检查 Node 和 npm 是否已正确安装：

```bash

node --version
npm --version
```

如果能看到版本号，恭喜你，nvm 和 Node 已经成功在 WSL 中安家了！

### 💡 额外小贴士

- **避免在 Windows 文件系统（`/mnt/c/`）下使用 nvm**：nvm 依赖符号链接，在跨文件系统时可能出问题。建议始终在你的 WSL 家目录（`/home/你的用户名`）下操作。
    
- **安装速度慢？** 可以尝试更换 npm 镜像（但 nvm 本身下载 Node 二进制文件是从官方源，可能受网络影响，国内用户可考虑设置代理或使用镜像）。
    
- **如果遇到权限问题**：永远不要用 `sudo` 配合 nvm 使用，nvm 是为普通用户设计的，所有安装都在 `~/.nvm` 目录下，不会污染系统全局。
