### 💡 第一步：准备工作

动手前，先把下面的东西准备好，可以避免中途卡壳：

- **OpenClaw环境**：确保 OpenClaw 已成功部署且能正常运行。
    
- **大模型API密钥**：配置好大模型（如阿里云百炼）的 API Key，这是智能对话的基础。
    
- **确保端口畅通**：OpenClaw 默认使用 `18789` 端口通信，记得在防火墙或云服务器的安全组里放行它。
    
- **飞书账号权限**：你需要一个拥有开发者权限的飞书账号，以便创建企业自建应用。
    

### 🤖 第二步：在飞书开放平台创建并配置机器人

这是让机器人在飞书里“有个身份”的过程。

1. **创建企业自建应用**
    
    - 登录[飞书开放平台](https://open.feishu.cn/)，进入开发者后台。
        
    - 点击**创建企业自建应用**，给你的机器人起个好听的名字（比如“我的AI助手”），完成创建。
        
2. **添加机器人能力**
    
    - 在应用详情页左侧，找到并点击**添加应用能力**，然后为你的应用开启**机器人**功能。
        
3. **获取核心凭证（关键！）**
    
    - 在左侧菜单进入**凭证与基础信息**，把这里的 **`App ID`** 和 **`App Secret`** 复制下来并**妥善保存**，下一步会用到。
        
4. **开通必要权限**
    
    - 进入**权限管理**，使用“批量导入/导出权限”功能，把下面的 JSON 代码粘贴进去，然后确认开通。这能让机器人拥有收发消息、读取信息的基本能力。
```
        {
          "scopes": {
            "tenant": [
              "im:message",
              "im:message.group_at_msg:readonly",
              "im:message.p2p_msg:readonly",
              "im:message:send_as_bot",
              "im:resource"
            ],
            "user": []
          }
        }
```
        
5. **配置事件订阅（另一关键点）**
    
    - 进入**事件与回调**页面。
        
    - 先**不要进行任何配置**，直接点击**保存**（如果提示未建立长连接，请忽略，这是正常的）。
        
    - 保存后，点击**添加事件**，搜索并添加 **`im.message.receive_v1`**（接收消息事件）。
        
    - **回到订阅方式**，选择 **“使用长连接接收事件”**，再次点击**保存。这样做可以省去配置公网回调地址的麻烦。
        
6. **发布应用**
    
    - 进入**版本管理与发布**，创建一个版本（如 `1.0.0`），设置好应用的可见范围（比如“所有员工”或仅限你自己），然后点击**确认发布**。应用只有发布后才能被搜索和使用。
        

### 🔗 第三步：在 OpenClaw 中完成对接

现在，我们让 OpenClaw 认识刚才创建好的飞书机器人。

1. **安装飞书插件（如果是手动部署）**
    
    - 如果你不是通过阿里云一键部署的 OpenClaw，可能需要手动安装飞书插件。打开命令行工具，执行以下命令：
        
```
        openclaw plugins install @openclaw/feishu
```
        
        
        
        _小提示：安装完插件最好先重启一下网关 `openclaw gateway restart`。_
        
2. **填入飞书应用凭证**
    
    - **方法一：命令行配置**（推荐，方便快捷）
        
```
        openclaw config set channels.feishu.appId "你保存的App ID"
        openclaw config set channels.feishu.appSecret "你保存的App Secret"
```
        
    - **方法二：控制台配置**：登录 OpenClaw 的 Web 控制台（通常是 `http://你的IP:18789`），找到“通道设置”或“渠道管理”，选择飞书，然后将 App ID 和 App Secret 填入对应位置。
        
3. **重启网关使配置生效**
    
    - 无论用哪种方式配置，最后都需要重启网关：
        
```
        openclaw gateway restart
```
        

### ✅ 第四步：验证与绑定

完成以上步骤，大功告成！现在来测试一下吧。

1. 打开飞书客户端，在通讯录或工作台找到你刚发布的机器人，给它发送一条消息（比如“你好”）。
    
2. 正常情况下，机器人会回复你一串**配对码**。
    
3. 将这个配对码通过 OpenClaw 提交，即可完成最终绑定.
```
   openclaw pairing approve feishu 你的配对码
```
    

### ✅ 验证一下

现在，回到飞书，给你的机器人发送一条消息（比如“你好”）。如果一切顺利，它应该能正常回复你，不再是配对码了。恭喜，配置成功！🎉
