### 第一步：在Windows中启用“虚拟机平台”

这是WSL 2正常运行的必要条件。你可以任选下面一种方法来完成：

**方法A：通过控制面板（图形界面，推荐新手）**

1. 在任务栏搜索框输入 **“启用或关闭Windows功能”** 并打开它。
    
2. 在弹出的窗口列表中，找到并 **勾选“虚拟机平台”** 和 **“Windows 虚拟机监控程序平台”**。
    
    - **注意**：如果找不到这个选项，可以先跳到本文的“第二步”，确认BIOS中的虚拟化技术已经开启，然后再回来操作。
        
3. 点击“确定”，系统会自动搜索并安装所需文件。
    
4. 安装完成后，根据提示**重启电脑**，让更改生效。
    

**方法B：通过PowerShell（命令行，更快捷）**

1. 右键点击开始菜单，选择 **“Windows PowerShell (管理员)”** 或 **“终端(管理员)”**。
    
2. 在打开的窗口中，输入以下命令并回车：
    
    powershell
    
```
    Enable-WindowsOptionalFeature -Online -FeatureName VirtualMachinePlatform
```
    
3. 命令执行完毕后，同样需要**重启电脑**。
    

### 第二步：在BIOS/UEFI中开启硬件虚拟化

如果在第一步中找不到“虚拟机平台”选项，或者重启后WSL 2仍无法使用，就需要检查这一步了。这是从硬件层面开启支持。

操作步骤如下：

1. **进入BIOS/UEFI设置界面**：
    
    - 重启电脑。
        
    - 在电脑启动、出现Windows徽标之前，反复按特定的**快捷键**。不同品牌的电脑快捷键不同，通常是 `F2`、`Del`、`F10` 或 `F12`。开机画面上通常会有短暂的提示，比如 `Press F2 to enter Setup`。
        
2. **找到并开启虚拟化技术选项**：
    
    - 进入BIOS界面后，使用键盘方向键进行导航。找到并进入 **“Advanced”**（高级）、**“CPU Configuration”**（CPU配置）或 **“Security”**（安全）等选项卡。具体名称可能因主板而异。
        
    - 在菜单中，寻找以下名称之一的选项，并将其状态设置为 **“Enabled”**（启用）：
        
        - **Intel Virtualization Technology** (常用于Intel处理器)
            
        - **VT-x** (Intel虚拟化技术的简称)
            
        - **AMD-V** (常用于AMD处理器)
            
        - **SVM Mode** (AMD处理器上的另一种常见叫法)
            
3. **保存并退出**：
    
    - 按 `F10` 键通常会弹出“保存并退出”的提示。选择 **“Yes”** 或 **“OK”** 确认。电脑会自动重启。
        

### 第三步：验证是否成功

重启电脑后，可以快速检查一下虚拟化功能是否已成功开启。

1. 按下 `Ctrl + Shift + Esc` 打开**任务管理器**。
    
2. 点击顶部的 **“性能”** 标签页。
    
3. 在左侧选择 **“CPU”**。
    
4. 查看右下角，如果 **“虚拟化”** 一栏显示 **“已启用”**，就说明所有设置都成功了。

### 第四步：安装Ubuntu-24.04
1、powershell下运行以下指令
```
wsl --install -d Ubuntu-24.04
```
2、如果没有显示正在下载Ubuntu-24.04，而是显示下载子系统，则等待下载安装完成后再执行一遍。

### 第五步：迁移Ubuntu-24.04到其他盘符
将C盘的wsl迁移至其他盘符，默认下载到C盘，并且大小可达数十G，需要手动迁移到其他盘符。
### 🚀 方法一：经典的导出/导入法 (最推荐)

这个方法非常可靠，适用于所有WSL版本。我们以你的 `Ubuntu-24.04` 为例进行操作。

1. **终止所有WSL进程**：首先，以管理员身份打开 **PowerShell**，运行以下命令来确保没有WSL实例在运行，避免文件占用。
    
    powershell
    
    wsl --shutdown
    
2. **导出当前的Ubuntu系统**：在PowerShell中，将你的整个Ubuntu系统打包成一个`tar`文件，并保存到你方便找到的地方（例如D盘的`wsl-backup`文件夹）。
    
    powershell
    
    wsl --export Ubuntu-24.04 D:\wsl-backup\ubuntu-24.04.tar
    
    - 命令执行后，你会看到D盘指定路径下生成了一个`.tar`文件，这就是你系统的完整备份。
        
3. **从C盘注销原系统**：导出成功后，就可以安全地将C盘里的原系统“卸载”了，这会立即释放被占用的C盘空间。
    
    powershell
    
    wsl --unregister Ubuntu-24.04
    
    **注意**：此操作会删除C盘的原系统，但上一步生成的备份文件是安全的。
    
4. **将系统导入到新位置**：现在，我们将备份文件“恢复”到你想要的目标位置（比如D盘的`WSL\Ubuntu-24.04`目录）。
    
    powershell
    
    wsl --import Ubuntu-24.04 D:\WSL\Ubuntu-24.04 D:\wsl-backup\ubuntu-24.04.tar --version 2
    
    - 这条命令中的三个关键信息是：发行版名称 `Ubuntu-24.04`、你想要存放的新路径 `D:\WSL\Ubuntu-24.04`，以及上一步的备份文件路径.
        
5. **恢复你的默认用户（关键一步）**：通过`--import`方式迁移的系统，默认会以`root`用户登录。你需要将它恢复为你之前使用的普通用户名。
    
    powershell
    
    # 请将 <你的用户名> 替换为你之前设置的用户名
    Ubuntu-24.04.exe config --default-user <你的用户名>
    
    例如，如果你的用户名是 `john`，则命令为 `Ubuntu-24.04.exe config --default-user john`。
    

### 🎯 方法二：更简洁的 `--move` 命令法 (需较新版本)

如果你使用的WSL版本较新（可以通过 `wsl --version` 查看），可以尝试这个更简洁的命令。它的好处是一步到位，无需生成中间备份文件.

1. 同样，首先在管理员PowerShell中运行 `wsl --shutdown` 关闭所有WSL进程。
    
2. 然后，直接运行迁移命令：
    
    powershell
    
    wsl --manage Ubuntu-24.04 --move D:\WSL\
    
    这个命令会将你的Ubuntu系统直接移动到D盘的 `WSL` 文件夹下。
    

**需要注意**：这个方法虽然简单，但并非所有WSL版本都支持。如果在执行时遇到错误提示，说明你的WSL版本暂不支持此功能，使用方法一即可。

### 💡 迁移后的小贴士

- **验证一下**：完成迁移后，重新启动Ubuntu，运行 `df -h /` 命令，查看文件系统的挂载点是否已经指向了你新的硬盘路径。
    
- **别忘了优化内存**：系统成功迁移到D盘后，通过 `C:\Users\你的用户名\.wslconfig` 文件限制WSL内存使用
